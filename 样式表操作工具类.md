## 一、背景与问题

### 1.1 `ss.Fill` 为 null 的根本原因

CorelDRAW X7 中，`StyleSheet.CreateStyleSet()` 返回的是一个**空样式集容器**。
此时 `ss.Fill`、`ss.Outline` 等属性全部返回 `Nothing`（C# 中为 `null`）。
直接赋值会抛出运行时错误（"对象变量或 With 块变量未设置"）：

```vba
' ✗ 错误写法（ss.Fill 此时为 null，会报运行时错误）
Dim ss As Style
Set ss = sheet.CreateStyleSet("", "我的样式集", True)
ss.Fill.Type = cdrUniformFillStyle          ' ← 运行时错误
ss.Fill.PrimaryColor.RGBAssign 0, 128, 255  ' ← 运行时错误
ss.Outline.Width = 0.5                      ' ← 运行时错误
```

**原因**：`StyleSet` 是多个单类别 `Style` 的容器。新建时容器为空，
`.Fill`、`.Outline` 等属性只有在对应的子样式被创建后才非 `null`。

---

### 1.2 唯一正确的修复路径

> ⚠️ **`Style`（样式集）对象没有 `CreateStyle` 方法**。网上流传的"先调用 `ss.CreateStyle("fill")` 激活子样式"的写法是错误的，该方法根本不存在。

唯一正确的做法是通过临时形状配合 `StyleSheet.CreateStyleSetFromShape` 捕获：

```vba
' ✓ 唯一正确写法：通过临时形状捕获
Dim tmp As Shape
Set tmp = ActiveLayer.CreateRectangle(-9999, -9999, -9998, -9998)
tmp.Fill.ApplyUniformFill CreateRGBColor(0, 128, 255)
tmp.Outline.Width = 0.5
sheet.CreateStyleSetFromShape tmp, "我的样式集", True
tmp.Delete
ActiveShape.ApplyStyle "我的样式集"
```

## 三、快速开始

### 1. 将 `CorelStyleSheetHelper.cs` 加入项目

将文件复制到 C# 项目中，命名空间为 `CorelDrawCOMAPI`。无需 NuGet 包，无需 COM 引用。

### 2. 引入命名空间

```csharp
using System.Runtime.InteropServices;
using CorelDrawCOMAPI;
```

### 3. 连接 CorelDRAW 并调用

```csharp
dynamic app = Marshal.GetActiveObject("CorelDRAW.Application");

// 创建一个蓝色填充 + 0.5 mm 黑色轮廓的样式集（正确处理 null 问题）
CorelStyleSheetHelper.CreateStyleSetViaShape(
    app, "品牌图形",
    fillR: 31, fillG: 73, fillB: 125,
    outlineWidthMm: 0.5,
    outlineR: 0, outlineG: 0, outlineB: 0);

// 应用到当前选中形状
CorelStyleSheetHelper.ApplyStyle(app.ActiveShape, "品牌图形");
```

## 四、API 速查

### 4.1 类别常量

```csharp
CorelStyleSheetHelper.CategoryFill      // "fill"
CorelStyleSheetHelper.CategoryOutline   // "outline"
CorelStyleSheetHelper.CategoryCharacter // "character"
CorelStyleSheetHelper.CategoryParagraph // "paragraph"
CorelStyleSheetHelper.CategoryFrame     // "frame"
```

### 4.2 方法一览

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `CreateFillStyle(app, name, r, g, b)` | `dynamic`（可 null） | 创建 RGB 纯色填充样式 |
| `CreateOutlineStyle(app, name, widthMm, r, g, b)` | `dynamic`（可 null） | 创建 RGB 轮廓样式 |
| `CreateCharacterStyle(app, name, font, size, bold, italic)` | `dynamic`（可 null） | 创建字符样式 |
| `CreateStyleSetViaShape(app, name, fillR/G/B, outlineW, outlineR/G/B)` | `dynamic`（可 null） | 通过临时形状创建样式集（唯一正确方式） |
| `CreateStyleSetFromShape(app, shape, name)` | `dynamic`（可 null） | 从现有形状捕获为样式集 |
| `CreateStyleFromShape(app, shape, category, name)` | `dynamic`（可 null） | 从现有形状捕获单类别样式 |
| `FindStyle(app, name)` | `dynamic`（可 null） | 按名称查找样式或样式集 |
| `ApplyStyle(shape, styleName)` | `bool` | 将样式应用到形状 |
| `ApplyStyleToShapes(shapes, styleName)` | `int` | 批量应用样式，返回成功数量 |
| `DeleteStyle(app, name)` | `bool` | 删除样式或样式集 |
| `RenameStyle(app, oldName, newName)` | `bool` | 重命名样式或样式集 |
| `GetStyleNames(app)` | `IReadOnlyList<string>` | 获取所有顶层样式名称列表 |
| `GetStyleSetNames(app)` | `IReadOnlyList<string>` | 获取所有顶层样式集名称列表 |
| `ExportStyles(app, filePath)` | `bool` | 导出样式表到 `.cdss` 文件 |
| `ImportStyles(app, filePath, merge)` | `bool` | 从 `.cdss` 文件导入样式表 |

## 五、分场景详解与示例

### 5.1 创建填充样式

```csharp
dynamic app = Marshal.GetActiveObject("CorelDRAW.Application");

// 创建一个名为"品牌蓝"的纯色填充样式
dynamic st = CorelStyleSheetHelper.CreateFillStyle(app, "品牌蓝", 31, 73, 125);
if (st != null)
    Console.WriteLine($"已创建填充样式：{(string)st.Name}");

// 应用到当前形状（只更改填充，轮廓保持不变）
CorelStyleSheetHelper.ApplyStyle(app.ActiveShape, "品牌蓝");
```

等价 VBA：
```vba
Dim st As Style
Set st = sheet.CreateStyle("fill", "", "品牌蓝", True)
st.Fill.Type = cdrUniformFillStyle
st.Fill.PrimaryColor.RGBAssign 31, 73, 125
ActiveShape.ApplyStyle "品牌蓝"
```

---

### 5.2 创建轮廓样式

```csharp
// 创建 0.5 mm 黑色轮廓样式
CorelStyleSheetHelper.CreateOutlineStyle(
    app, "细黑线",
    widthMm: 0.5,
    r: 0, g: 0, b: 0);

// 同时应用填充 + 轮廓（两个独立样式叠加）
CorelStyleSheetHelper.ApplyStyle(app.ActiveShape, "品牌蓝");
CorelStyleSheetHelper.ApplyStyle(app.ActiveShape, "细黑线");
```

### 5.3 创建字符样式

    fontName: "微软雅黑",
    sizePoints: 14,
    bold: true,
    italic: false);

// 应用到文字形状

### 5.4 创建样式集（通过临时形状，推荐）

```csharp
// 创建蓝色填充 + 0.5 mm 黑色轮廓的样式集
// 内部原理：先创建临时矩形并配置外观，再 CreateStyleSetFromShape 捕获，最后删除临时形状
dynamic ss = CorelStyleSheetHelper.CreateStyleSetViaShape(
    app, "品牌图形",
    fillR: 31, fillG: 73, fillB: 125,
    outlineWidthMm: 0.5,
    outlineR: 0, outlineG: 0, outlineB: 0);

if (ss != null)
    CorelStyleSheetHelper.ApplyStyle(app.ActiveShape, "品牌图形");
```

---

### 5.6 从现有形状捕获样式/样式集

```csharp
// 先配置好形状外观
dynamic shape = app.ActiveShape;
shape.Fill.ApplyUniformFill(app.CreateRGBColor(0, 200, 100));
shape.Outline.Width = 1.5;

// 捕获为样式集（包含填充 + 轮廓）
CorelStyleSheetHelper.CreateStyleSetFromShape(app, shape, "绿色宽边");

// 只捕获填充
CorelStyleSheetHelper.CreateStyleFromShape(
    app, shape, CorelStyleSheetHelper.CategoryFill, "绿色填充");

// 只捕获轮廓
CorelStyleSheetHelper.CreateStyleFromShape(
    app, shape, CorelStyleSheetHelper.CategoryOutline, "宽边轮廓");
```

### 5.7 应用样式到形状

```csharp
// 单个形状
bool ok = CorelStyleSheetHelper.ApplyStyle(app.ActiveShape, "品牌图形");

### 5.8 批量应用

```

## 六、对象模型说明

```
Application
└── ActiveDocument
    └── StyleSheet               ← CorelStyleSheetHelper 的操作入口
        ├── Styles               ← 顶层单类别样式集合（IsStyleSet = False）
        ├── StyleSets            ← 顶层样式集集合（IsStyleSet = True）
        ├── AllStyles            ← 全部样式（含派生子样式）
        ├── AllStyleSets         ← 全部样式集
        └── ObjectDefaults       ← 内置默认样式集（"Default Graphic" 等）

Style（IsStyleSet = False，单类别）
├── CategoryName → "fill" / "outline" / "character" / "paragraph" / "frame"
├── Fill         → StyleFill（Category = "fill" 时非 null）
├── Outline      → StyleOutline（Category = "outline" 时非 null）
├── Character    → StyleCharacter（Category = "character" 时非 null）
├── Paragraph    → StyleParagraph（Category = "paragraph" 时非 null）
└── Frame        → StyleFrame（Category = "frame" 时非 null）

Style（IsStyleSet = True，样式集）
├── Styles       → 子样式集合（每个子样式是单类别 Style，由 CreateStyleSetFromShape 自动创建）
├── Fill         → StyleFill（仅当 Styles 中含 fill 子样式时非 null）
├── Outline      → StyleOutline（仅当 Styles 中含 outline 子样式时非 null）
└── ...
```

**关键结论**：
- 单类别 `Style`（`CreateStyle("fill")`）创建后，`.Fill` 立即非 null。
- 样式集（`CreateStyleSet()`）创建后，`.Fill`、`.Outline` 均为 **null**。
- `Style`（样式集）对象**没有 `CreateStyle` 方法**，无法在容器上直接添加子样式。
- 必须通过 `CreateStyleSetFromShape` 从已格式化的形状捕获，才能激活这些属性。

## 七、返回值始终安全

所有方法内部均包含完整的 `try/catch`，不会向外抛出 COM 异常。

| 方法类型 | 无法执行时 |
|---------|-----------|
| 返回 `dynamic` 的方法 | 返回 `null`，调用前需 `!= null` 检查 |
| 返回 `bool` 的方法 | 返回 `false` |
| 返回 `int` 的方法 | 返回 `0` |
| 返回 `IReadOnlyList` 的方法 | 返回空列表（非 null） |

## 八、常见问题

**Q：为什么不能用 `ss.CreateStyle("fill")` 激活 StyleSet 的子属性？**  
A：`Style`（样式集）对象根本没有 `CreateStyle` 方法，该方法只属于 `StyleSheet`。
`CreateStyleSet` 返回的是一个空容器，唯一能让 `.Fill`/`.Outline` 变为非 null 的方式
是通过 `CreateStyleSetFromShape` 从已有形状整体捕获。

**Q：应用样式集后直接修改形状的 `Fill`，是否会影响其他使用该样式的形状？**  
A：不会。在形状上直接修改属性（如 `s.Fill.ApplyUniformFill`）会产生**局部覆盖**，
使该形状脱离样式控制。之后修改样式，该属性不再跟随样式更新。

**Q：样式集能否只包含填充（不包含轮廓）？**  
A：可以。使用 `CreateStyleSetViaShape` 时，传入 `outlineWidthMm: 0` 即跳过轮廓设置，
临时形状只捕获填充属性。

**Q：如何修改已存在的样式？**  
```csharp
dynamic st = CorelStyleSheetHelper.FindStyle(app, "品牌蓝");
if (st != null)
    st.Fill.PrimaryColor.RGBAssign(220, 50, 50); // 改为红色，所有应用该样式的形状同步更新
```

**Q：`FindStyle` 能同时查找普通样式和样式集吗？**  
A：可以。`StyleSheet.FindStyle(Name)` 在所有样式（包括样式集）中搜索名称，
本类的 `FindStyle` 方法直接委托给它。
